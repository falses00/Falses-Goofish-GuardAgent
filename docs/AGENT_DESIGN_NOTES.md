# Agent 设计优化说明

## 本轮采用的设计原则

- **模型不直接拥有业务底线**：价格底线、历史承诺和买家最高出价由代码和数据库控制。
- **路由先规则后模型**：高确定性的咨询和议价意图先用关键词与正则命中，降低无谓模型调用。
- **事实检索优先于自由生成**：商品事实来自 `data/product_info.json`，未命中时不伪装成命中。
- **可观测优先**：每轮回复都生成 `AgentTrace`，让调试从“看最终回复”变成“看决策链路”。
- **测试覆盖失败路径**：把错误折扣配置、规格数字误判、历史报价回升等真实风险写进测试。

## 当前边界

- 当前知识库是轻量 JSON 检索，不是向量数据库 RAG。
- 当前路由主要依赖规则和分类 Prompt，没有引入独立工作流引擎。
- 当前真实平台连接仍依赖 Cookie，Cookie 失效和平台风控不是本项目能完全解决的问题。

## 后续可继续优化

- 为 `AgentTrace` 增加 JSONL 文件落盘，形成可回放评测集。
- 为不同商品维护独立策略文件，支持多商品动态底价和发货规则。
- 引入离线 eval：固定买家输入集，比较不同 prompt / model 的回复质量与护栏通过率。
- 增加人工接管 Web 面板，展示会话状态、护栏命中、待处理订单和异常消息。
